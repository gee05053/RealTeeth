# RealTeeth (리얼티쓰)

현재 위치 또는 검색한 지역의 날씨를 확인하고, 자주 보는 지역을 최대 6개 까지 북마크로 저장할 수 있습니다.

---

## 실행 방법

### 사전 요구 사항

- Node.js 18+
- Yarn 4.12.0

### 환경 변수 설정

프로젝트 루트에 `.env` 파일을 생성하고 아래 키를 입력합니다.

```env
VITE_KAKAO_REST_API_KEY=카카오 REST API 키
VITE_WEATHER_API_KEY=기상청 API 키
```

- **카카오 REST API 키**: [Kakao Developers](https://developers.kakao.com)에서 애플리케이션 생성 후 발급
- **기상청 API 키**: [공공데이터포털](https://www.data.go.kr)에서 `기상청 단기예보((구)동네예보) 조회서비스` 신청 후 발급

### 개발 서버 실행

```bash
yarn install
yarn dev
```

### 프로덕션 빌드

```bash
yarn build
yarn preview
```

## 구현 기능

### 날씨 조회

- 브라우저 GeolocationAPI를 사용하여, 현재 위치의 날씨를 자동으로 불러옵니다. (위치 권한 동의 필요)
- 현재 기온, 하늘 상태 혹은 강수 형태를 표시합니다.
- 당일 최저/최고 기온을 표시합니다.
- 24시간 시간대별 예보를 가로 스크롤 형태로 표시합니다. (시간, 날씨 아이콘, 기온)

### 지역 검색

- 지역명으로 검색하면 관련 행정구역 목록이 드롭다운으로 표시됩니다.
- 검색어와 완전히 일치하는 항목을 부분 일치 항목보다 우선 노출하고, 동점일 경우 가나다 순으로 정렬합니다.
- 키보드(↑↓ 탐색, Enter 선택, Escape 닫기)와 마우스 모두 지원합니다.
- 검색 입력에 300ms 디바운스를 적용해 불필요한 연산을 줄입니다.

### 북마크

- 현재 조회 중인 위치를 북마크에 추가하거나 제거할 수 있습니다. (최대 6개)
- 북마크마다 사용자 지정 별칭을 설정할 수 있습니다.
- 드래그 앤 드롭으로 북마크 순서를 변경할 수 있습니다.
- 북마크를 클릭하면 해당 위치의 상세 날씨 페이지로 이동합니다.
- 데이터는 `localStorage`에 저장되어 새로고침 후에도 유지됩니다.

### UI/UX

- 시간대에 따라 배경 그라데이션이 변합니다. (새벽 / 아침 / 낮 / 저녁 / 밤)
- 모바일 우선 반응형 레이아웃입니다.

## 기술적 의사결정 및 이유

### TanStack Query로 서버 상태 관리

**이유**: 날씨 API 응답을 컴포넌트 로컬 state로 관리하면 동일 데이터를 중복 요청하거나 로딩/에러 상태를 각 컴포넌트에서 개별 처리해야 합니다. TanStack Query를 사용해 캐싱, 로딩/에러 상태, 재시도 로직을 선언적으로 처리했습니다.

---

### 북마크 저장소로 localStorage 사용

**이유**: 사용자가 브라우저 탭을 닫아도 설정한 북마크 데이터를 유지하기 위해 로컬 스토리지를 사용했습니다. 커스텀 이벤트를 활용해 탭 간 상태 동기화도 처리했습니다.

---

### @dnd-kit으로 북마크 드래그 앤 드롭 순서 변경 구현

**이유**: 북마크는 최대 6개까지 저장할 수 있는데, 순서 변경 기능이 없으면 기존 북마크를 삭제하고 다시 추가해야 합니다. 이 불편함을 해결하기 위해 @dnd-kit을 사용해 북마크 카드의 순서를 드래그로 변경할 수 있도록 구현했습니다.
